Install SQL Server on VPS

DunLan

数据库

发布于：2020年8月13日

快速在服务器上安装SQL Server

准备工作

设置本机防火墙

打开本机的防火墙，在入站规则处新建一个端口类型的入站规则，协议为 TCP，端口为 1433 或其他。

设置服务器入站端口规则

本文以 Microsoft Azure 云（非世纪互联版）Ubuntu 虚拟机为例，创建一个 VPS 或者叫虚拟机，在网络处添加一个入站端口规则。（详见 此文：[Azure SQL 文档] 及下图）

PS：不要参照上面的文档中说的 在服务列表中选择"MS SQL" 它在演你！（不要问我是怎么知道的😭） 按下图设置就好

然后通过SSH登陆，并打开一个终端。

PS：使用 Windows Terminal 的同学可以直接在设置文件中的 profiles 或者 list 列表中添加

{
    "guid": "{b23e9e35-82da-4c33-8694-02566110848e}",
    "name": "name",
    "commandline": "powershell.exe ssh <用户名，一般是root>@<服务器IP地址>",
}

其中 UUID 可以通过 这个 网站生成

然后即可通过密码登陆了

设置 VPS 防火墙规则（可选）

无论是否开启 Ubuntu 的防火墙，实测，对连接数据库无影响，但是强迫症嘛，当然是要搞滴(不是)为了保险起见，还是设置一下。

防火墙应该是默认关闭的。

首先执行 sudo ufw allow 22/tcp 命令，22是 SSH 登陆需要的端口

再执行 sudo ufw allow 1433/tcp 命令，添加数据库端口

执行 sudo ufw enable 打开防火墙

sudo ufw status 查看端口规则

其他命令参考：

sudo ufw disable                  //关闭防火墙
sudo ufw reload                   //重启防火墙，使配置生效
sudo ufw default deny             //拒接所有外来访问，本机能正常访问外部
sudo ufw delete allow 22          //关闭21端口
sudo ufw allow from 192.168.121.1 //指定ip为192.168.121.1的计算机操作所有端口
sudo ufw allow from 192.168.121.2 to any port 3306 //开放指定ip为__的计算机访问本机的3306端口

连接

此处采用 DataGrip 做例子

Host 处填 VPS 的公共 IP 地址

密码是数据库的密码

Port 即准备工作中开放的端口

测试连接通过后即可愉快的使用啦
